# 🟢 موقع نادي القيروان — منصة موسمية متكاملة

موقع متكامل (Full-Stack) لمتابعة طلاب نادي القيروان، نقاطهم، حضورهم، ومجموعاتهم،
بهوية بصرية مستوحاة من شعار النادي.

**التقنيات المستخدمة:**
- الواجهة: HTML / CSS / JavaScript (EJS Templates)
- الخادم: Node.js + Express
- قاعدة البيانات: **MySQL**
- الجلسات: express-session (لتسجيل دخول المشرفين)
- الباركود: QR Code (qrcodejs للتوليد + jsQR للقراءة بالكاميرا)
- الاختبارات الآلية: Jest + Supertest (193 اختبار، راجع `tests/README.md`)
- النشر: Railway

## ✅ الميزات الحالية

### الصفحة الرئيسية
نبذة تعريفية عن النادي، عدد المجموعات والطلاب، عدد أسابيع الموسم وأيامه، **أيام
النادي الثلاثة الثابتة (الاثنين/الثلاثاء/الأربعاء)** مع تاريخ بداية الموسم، وقائمة
أفضل 10 طلاب.

### نظام الجلسات الثابتة (9 جلسات بتواريخ محددة)
الحضور لم يعد بتاريخ حر، بل مرتبط بـ **9 جلسات ثابتة بتواريخ دقيقة مسبقة التحديد**:
3 أسابيع × 3 أيام (الاثنين، الثلاثاء، الأربعاء)، تبدأ من **13 يوليو 2026**. عند
مسح المشرف لباركود طالب، يُسجَّل الحضور تلقائياً لـ"جلسة اليوم" إن طابقها التاريخ
الفعلي، أو لأقرب جلسة قادمة لم تُعقد بعد. الجلسات التسع مُعرَّفة مسبقاً في
`config/schema.sql` ولا تُنشأ ديناميكياً.

### بوابة ولي الأمر
بحث فوري باسم الطالب، عرض ترتيبه **داخل مجموعته** (وليس على مستوى النادي)، وجدول
حضور منظَّم بالأسابيع الثلاثة يعرض كل الجلسات التسع — حتى التي لم تُعقد بعد (تظهر
بحالة "لم تُعقد بعد").

### المجموعات (مقسَّمة لفئتين)
المجموعات الست موزَّعة على قسمين: **الفئة الصغرى** و**الفئة العليا** (3 مجموعات
لكل قسم)، وكل مجموعة مرتبة داخل قسمها حسب إجمالي نقاطها.

### لوحة المشرفين
- رمز الدخول الافتراضي: **`991`**
- **بحث سريع وفوري** بالاسم يفلتر جدول الطلاب مباشرة أثناء الكتابة (مهم لعدد كبير من الطلاب)
- إضافة أو خصم نقاط لأي طالب في أي برنامج (معرفي/رياضي/ثقافي/مبادرات)
- تسجيل حضور يدوي لجلسة محددة يختارها المشرف من قائمة الجلسات التسع
- تسجيل حضور تلقائي عبر مسح باركود الطالب بالكاميرا
- جدول يعرض كل الطلاب وباركود كل واحد للطباعة

### الهوية البصرية
التصميم بالكامل (الألوان، الأزرار، البطاقات) مبني على ألوان شعار النادي: أصفر،
تركواز، برتقالي، بنفسجي، على خلفية كحلية. الشعار نفسه ظاهر بهيدر كل صفحة.

## ⚙️ التشغيل محلياً (على جهازك)

### 1) تثبيت MySQL محلياً
نزّل وثبّت MySQL Community Server من الموقع الرسمي إذا لم يكن مثبتاً عندك:
https://dev.mysql.com/downloads/mysql/

أنشئ قاعدة بيانات جديدة (عبر MySQL Workbench أو سطر الأوامر):
```sql
CREATE DATABASE qayrawan_club CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

> مهم: لازم تستخدم `utf8mb4` (وليس `utf8` العادي) لأن النصوص عربية، و `utf8` القديم بـ MySQL لا يدعم كل الأحرف بشكل كامل.

### 2) تثبيت حزم المشروع
```bash
cd qayrawan-club
npm install
```

### 3) ضبط متغيرات البيئة
```bash
cp .env.example .env
```
ثم افتح ملف `.env` وعدّل `DATABASE_URL` ليطابق بيانات اتصالك المحلية، مثلاً:
```
DATABASE_URL=mysql://root:كلمة_السر@localhost:3306/qayrawan_club
```

### 4) إنشاء الجداول وتعبئتها ببيانات تجريبية
```bash
npm run seed
```
هذا الأمر يُنشئ كل الجداول تلقائياً من `schema.sql` (شاملةً الجلسات التسع تلقائياً)
ثم يضيف 30 طالباً تجريبياً موزعين على 6 مجموعات (3 بكل فئة)، مع حضور تجريبي لكل
الجلسات التسع الفعلية.

### 5) تشغيل الموقع
```bash
npm run dev
```
(يستخدم nodemon لإعادة التشغيل التلقائي عند أي تعديل)

أو للتشغيل العادي:
```bash
npm start
```

افتح المتصفح على: **http://localhost:3000**

## 🚀 النشر على Railway

### الخطوة 1: رفع المشروع إلى GitHub
```bash
git init
git add .
git commit -m "نسخة أولى من موقع نادي القيروان"
git branch -M main
git remote add origin https://github.com/اسم_حسابك/qayrawan-club.git
git push -u origin main
```
> تأكد أن ملفي `.env` و`.env.test` **غير مرفوعين** (موجودان ضمن `.gitignore` بالفعل).

### الخطوة 2: إنشاء مشروع على Railway
1. اذهب إلى https://railway.app وسجّل دخول (يفضّل عبر GitHub).
2. اضغط **New Project** → **Deploy from GitHub repo**.
3. اختر مستودع `qayrawan-club` الذي رفعته.

### الخطوة 3: إضافة قاعدة بيانات MySQL
1. داخل نفس المشروع على Railway، اضغط **+ New** → **Database** → **Add MySQL**.
2. Railway سينشئ قاعدة البيانات تلقائياً ويولّد متغير `DATABASE_URL` بنفسه (بصيغة `mysql://...`).
3. اربط هذا المتغير بخدمة الموقع: من تبويب الموقع (Service) → **Variables** → تأكد أن `DATABASE_URL` مرتبط بقاعدة MySQL (Railway يتيح هذا عبر "Reference Variable"، وإن لم يحدث تلقائياً أضفه يدوياً بنفس القيمة الموجودة في خدمة قاعدة البيانات).

### الخطوة 4: ضبط باقي متغيرات البيئة
من تبويب الموقع (Service) → **Variables** → أضف:
```
SESSION_SECRET=ضع_قيمة_عشوائية_طويلة_وسرية
SUPERVISOR_ACCESS_CODE=991
NODE_ENV=production
```
> لا تضع `PORT` يدوياً؛ Railway يضبطه تلقائياً.

### الخطوة 5: تعبئة قاعدة البيانات على Railway
بعد أول نشر، تحتاج لتشغيل سكربت `seed` مرة واحدة لإنشاء الجداول والبيانات الأولية:

1. ثبّت Railway CLI:
   ```bash
   npm i -g @railway/cli
   ```
2. سجّل الدخول واربط المشروع:
   ```bash
   railway login
   railway link
   ```
3. شغّل أمر التهيئة على بيئة Railway:
   ```bash
   railway run npm run seed
   ```

### الخطوة 6: التأكد من إعدادات التشغيل
Railway يكتشف ملف `package.json` تلقائياً ويشغّل `npm start`. لا حاجة لإضافة Procfile، لكن إن رغبت بالتوضيح يمكنك إضافة ملف `railway.json` (موجود مسبقاً في المشروع).

### الخطوة 7: فتح الموقع
بعد انتهاء النشر، Railway يعطيك رابطاً مباشراً مثل:
```
https://qayrawan-club.up.railway.app
```
يمكنك أيضاً ربط دومين خاص بك من تبويب **Settings → Domains**.

## 🛠️ أوامر مفيدة

| الأمر | الوظيفة |
|---|---|
| `npm run dev` | تشغيل محلي مع إعادة تشغيل تلقائي |
| `npm start` | تشغيل عادي (للإنتاج) |
| `npm run seed` | إنشاء/إعادة إنشاء الجداول وتعبئتها ببيانات تجريبية |
| `npm test` | تشغيل كل الاختبارات الآلية مرة واحدة |
| `npm run test:watch` | تشغيل الاختبارات بوضع المراقبة المستمر |
| `npm run test:coverage` | تشغيل الاختبارات مع تقرير تغطية الكود |

⚠️ تحذير: تشغيل `npm run seed` يحذف كل الجداول الحالية ويعيد إنشاءها من الصفر (`DROP TABLE` ثم إنشاء). لا تشغّله على بيانات حقيقية تريد الاحتفاظ بها. نفس التحذير ينطبق على `npm test` لكن **فقط على قاعدة بيانات الاختبار المنفصلة** (`qayrawan_club_test`)، وهي محمية بفحص أمان يرفض التشغيل إن لم تحتو `_test` باسمها.

## 📝 ملاحظات فنية حول استخدام MySQL

- جدول المجموعات اسمه `groups` بقاعدة البيانات، وهذه كلمة محجوزة في MySQL 8، لذلك تجدها مكتوبة دائماً بـ backticks: `` `groups` `` في كل الاستعلامات. لا تحذف هذه العلامات إذا عدّلت الكود.
- جدول `groups` يحتوي عمود `category` بقيمتين فقط (`ENUM`): "الفئة الصغرى" و"الفئة العليا".
- جدول `sessions` يحتوي الجلسات التسع الثابتة للموسم (مُدرَجة تلقائياً ضمن `schema.sql`)، وجدول `attendance` يرتبط بـ `session_id` (وليس بتاريخ حر كما كان سابقاً).
- حالة الحضور (`status`) مخزّنة كـ `ENUM('حاضر', 'متأخر', 'غايب')` لضمان عدم إدخال قيم غير صحيحة.
- التواريخ تُسترجع كنص بسيط (`YYYY-MM-DD`) بدل كائن JavaScript Date، بسبب خيار `dateStrings: true` في `config/db.js`.
- التشفير المستخدم في كل الجداول هو `utf8mb4` لدعم النصوص العربية بشكل كامل.
